## 2. 사업 개요

### 2.1 서비스명 및 종류

- **서비스명**: PawMate
- **서비스 종류**: 반려동물 보호자 커뮤니티 및 위치기반 매칭 모바일 앱
- **플랫폼**: iOS / Android (Flutter 기반 단일 코드베이스)
- **서비스 제공 지역**: 대한민국 전역
- **사업 유형**: 위치기반서비스사업(개인위치정보 이용) 신고 대상
  - 신고 대상 판단 근거: 이용자의 개인위치정보를 단말기 내에서만 처리하지 않고
    **사업자의 위치정보시스템(서버)으로 전송·저장·이용**하므로 신고 대상에 해당

### 2.2 서비스 목적

반려동물 보호자 간 지역 기반의 신뢰 네트워크를 구축하여 동반산책, 대리산책,
돌봄, 분양, 입양 등의 활동을 안전하게 교류할 수 있는 커뮤니티를 제공합니다.
이용자의 활동 지역 및 게시글 위치를 기준으로 인근의 게시물·시설물을
탐색·매칭할 수 있도록 위치정보를 활용합니다.

### 2.3 주요 기능

| 구분 | 기능 | 위치정보 활용 |
|---|---|---|
| 커뮤니티 | 게시글 작성/조회/검색 (6개 카테고리) | 게시글 위치 기준 지도 표시 |
| 매칭 | 지원-수락-약속-완료 흐름, 평가 시스템 | 지역 기반 필터 |
| 채팅 | 1:1 실시간 채팅 | (직접 이용 없음) |
| 소셜 | Pawing(팔로우), 하트(관심 게시글) | (직접 이용 없음) |
| 지도 | 주변 게시글 및 반려동물 시설(병원·미용실 등) | 이용자 위치·게시글 좌표 표시 |
| 지역 인증 | 활동 지역 인증(2주 주기) | 인증 시 GPS 위치 확인 |
| 알림 | 인앱 알림 + FCM/APNs 푸시 | (직접 이용 없음) |

### 2.4 요금 체계 및 수익 구조

- **이용 요금**: 전면 무료. 앱 내 유료 기능, 구독, 인앱결제 없음
- **광고**: 게재하지 않음
- **결제·정산 기능**: 앱 내 결제, 송금, 정산, 수수료 수취 기능 일체 없음
- **이용자 간 대가 수수**: 대리산책·돌봄 등에서 이용자 간 대가가 필요한 경우
  당사자 간 합의에 따라 **앱 외부에서** 이루어지며, 사업자는 거래에 관여하지 않고
  중개 수수료를 수취하지 않음
- 향후 유료 기능·결제 기능 도입 시 관련 법령(전자상거래법 등) 검토 및 필요한 신고를 선행할 예정

### 2.5 분양·입양 게시판 운영 방식

- 이용자가 키우던 반려동물의 새 보호자를 찾거나(분양), 보호자를 찾는 동물을 입양하고자
  할 때 게시글을 통해 **이용자 간 직접 교류**하는 구조
- 앱은 게시글 작성·조회와 1:1 채팅의 장만 제공하며, 거래·계약·대금에 일체 관여하지 않음
- **운영 정책**:
  - 영리 목적의 반복적 분양 게시 금지 (동일 이용자의 게시 빈도 제한)
  - 동물생산업·동물판매업 등 영업자의 상업적 판매 게시 금지 (약관 명시)
  - 게시글 신고 기능 및 운영자 모니터링을 통한 위반 게시글 삭제·이용 제한
  - 분양·입양 카테고리 게시 시 관련 법령 준수 안내문 고지

## 3. 위치정보 수집 및 이용

### 3.1 수집하는 위치정보

- **개인위치정보 대상자**: 만 14세 이상 이용자
  - 가입 시 생년월일 입력 및 전화번호 인증을 통해 만 14세 미만의 가입을 차단
  - 만 14세 미만은 서비스 제공 대상이 아니므로 법정대리인 동의 절차 불요
    (향후 대상 확대 시 위치정보법 제18조·제19조에 따른 법정대리인 동의 절차 구축 예정)
- **수집 항목**: 이용자 단말기의 위·경도 좌표(GPS 기반), Wi-Fi/기지국 정보 기반 대략 좌표
- **수집 방식**: 위치기반서비스 이용약관 동의 및 단말기 위치 서비스 권한 허용 후,
  앱을 통해 단말기에서 수집 (최초 실행 시 동의 화면 제공)
- **정확도**: GPS 정상 수신 시 ±10m 내외, 실내/음영지역은 Wi-Fi·기지국 기반 대략 좌표 사용

### 3.2 위치정보 이용 목적

1. 이용자 활동 지역 등록 및 인증 (2주 주기 재인증)
2. 게시글 작성 시 위치 정보 기록 (실제 좌표는 서버 내부 처리 전용, 공개 좌표는 50~200m 랜덤 offset 적용)
3. 인근 게시글·시설물 지도 표시 및 반경 검색
4. 반려동물 관련 시설(병원, 미용실, 카페 등) 캐시 데이터 표시
5. 위치 급변(비현실적 이동) 감지를 통한 부정 이용 차단

### 3.3 위치정보사업자와의 관계

**본 사업자는 위치정보사업자(위치정보를 수집·제공하는 자)에 해당하지 않으며,
위치기반서비스사업자(수집된 위치정보를 이용하는 자)에 해당합니다.**

- 지도 및 좌표 관련 서비스: **네이버 지도 API**로부터 지오코딩·지도 타일·주변 시설 검색
  기능을 제공받아 이용 (지도 API는 네이버로 단일화)
- 이용자 단말기의 좌표: 단말기(iOS/Android) OS API를 통해 직접 수집
- 자체적으로 위치정보사업(위치정보 수집·제공)을 영위하지 않음

### 3.4 위치정보 보유 및 파기

| 데이터 | 보유 기간 | 근거 |
|---|---|---|
| 활동 지역 좌표 (users.latitude/longitude) | 회원 탈퇴 또는 동의 철회 시 지체 없이 파기 | 서비스 제공 (내부 방침) |
| 게시글 위치 좌표 (posts.actual_lat/lng) | 게시글 삭제 시 지체 없이 파기 | 서비스 제공 (내부 방침) |
| 위치 인증 이력 (location_verifications) | 6개월 후 파기 | 내부 방침 |
| 위치정보 이용·제공사실 확인자료 | 6개월간 보존 후 지체 없이 파기 | 위치정보법 제16조 제2항 |

- **동의 철회 시 처리**: 이용자가 위치정보 수집·이용 동의를 전부 또는 일부 철회한 경우,
  법정 보존 의무가 있는 이용·제공사실 확인자료를 제외한 개인위치정보를 지체 없이 파기
- **파기 방법**: 전자적 파일은 복구·재생이 불가능한 방법으로 영구 삭제
  (DB 레코드 삭제 및 백업 데이터는 백업 보존 주기 [[예: 30일]] 경과 시 자동 소멸)
- 회원 탈퇴 후에도 이용·제공사실 확인자료는 법정 기간(6개월) 동안 분리 보존 후 파기

## 4. 위치정보시스템 (주요 설비)

### 4.1 시스템 구성 개요

```
[이용자 단말기]           [백엔드 (Supabase)]              [외부 지도 API]
   ├ GPS/Wi-Fi/기지국    ├ PostgreSQL DB                 └ 네이버 지도 API
   ├ Flutter 앱          ├ PostgREST REST API               (지오코딩, 지도 타일,
   └ HTTPS/WSS           ├ Realtime (WebSocket)              시설 POI 검색)
                         ├ Storage (이미지)
                         └ Edge Function (푸시 발송)
```

### 4.2 주요 설비 상세

| 구분 | 설비 | 사양 / 서비스 | 설치 장소 |
|---|---|---|---|
| 서버 | Supabase Cloud (관리형 PostgreSQL 15) | 위치정보 저장·처리 | [[리전 확정 필수, 예: AWS ap-northeast-2 (서울)]] |
| DB | PostgreSQL 15 | 위치정보는 users, posts, location_verifications 테이블에 저장 | 상동 |
| 인증 | 커스텀 JWT (argon2id 비밀번호 해시) | 전화번호 OTP 기반 신원 확인 | — |
| SMS 발송 | Solapi (구 CoolSMS) HTTPS API | 전화번호 인증코드 | 국내 |
| 파일 저장 | Supabase Storage | 프로필/펫/게시글 이미지 | 상동(서버 리전) |
| 실시간 통신 | Supabase Realtime (WebSocket) | 채팅 메시지, 미읽음 반영 | 상동(서버 리전) |
| 푸시 알림 | FCM (Android) / APNs (iOS) | 백그라운드 알림 (위치정보 미포함) | 국외(미국 등) |
| 지도 API | 네이버 지도 API | 지오코딩, 시설 POI, 지도 표시 | 국내 |
| 클라이언트 | Flutter 3.x (Dart) | iOS 15+ / Android API 30+ | — |
| 관리 도구 | Supabase Dashboard | 운영자 접근 (2FA 적용) | — |

### 4.3 위치정보 흐름

1. 이용자 단말기에서 GPS/Wi-Fi/기지국 정보를 통해 좌표 획득
2. HTTPS(TLS 1.2+)로 Supabase PostgREST API에 전송
3. 서버는 실제 좌표(actual_lat/lng)를 저장하되 API 응답에서는 제외
4. 공개용 좌표(display_lat/lng)는 50~200m 랜덤 offset 적용 후 클라이언트에 전달
5. 위치 인증 시 활동 지역 좌표와 현재 GPS의 거리를 서버에서 비교

## 5. 위치정보 보호조치

### 5.1 관리적 보호조치

- **위치정보관리책임자 지정**: [[성명 / 직위]] — 위치정보 보호조치 총괄
  (위치정보법 시행령에 따른 지정, 개인정보보호책임자와 겸직 시 그 사실 명기)
- **개인정보보호책임자 지정**: [[성명 / 직위]] (책임자 연락처 별도 게시)
- **내부관리계획(관리지침) 수립**: 방송미디어통신위원회 고시
  **「위치정보의 관리적·기술적 보호조치 기준」**에 따라 위치정보 취급자·접근권한·안전조치 사항을 문서화
- **접근권한 관리**: 최소권한 원칙, 취급자별 계정 분리, 반기 1회 권한 재검토
- **취급자 교육**: 신규 취급자 배정 시 및 연 1회 정기 교육 실시
- **위치정보 보호 서약서**: 취급자 지정 시 서명·보관

### 5.2 기술적 보호조치

- **전송 구간 암호화**: 모든 API 호출을 HTTPS(TLS 1.2 이상)로 강제
- **저장 시 암호화**: Supabase Cloud의 스토리지 레벨 암호화(AES-256) 적용
- **인증·접근제어**:
  - 이용자 인증: 커스텀 JWT (Supabase 프로젝트 secret 서명, `sub`+`role` 클레임)
  - 행 단위 접근 제어: PostgreSQL Row Level Security (RLS) 정책으로 위치정보 열람 제한
  - 관리자 접근: 2FA 필수
- **비밀번호 저장**: argon2id 해시 (RFC 9106 기준), 원문 저장 금지
- **좌표 노출 최소화**:
  - 실제 좌표(actual_lat/lng)는 API 응답에서 제외
  - 공개 좌표(display_lat/lng)는 랜덤 offset(50~200m) 적용, 게시글마다 독립 offset
  - `is_location_hidden=true` 설정 시 공개 좌표도 미전송
- **기록 보관**:
  - 위치정보 이용·제공사실 확인자료(요청 시각, 이용자 식별정보, 처리 결과): 위치정보시스템에 자동 기록, 6개월간 보존 후 파기 (위치정보법 제16조 제2항)
  - 개인정보처리시스템 접속기록(관리자·취급자 접근 로그): 1년 이상 보관
    (개인정보의 안전성 확보조치 기준)
- **어뷰징 방지**: 1시간 내 비현실적 이동 감지 시 접근 차단, 실패 횟수 임계치 초과 시 일시 잠금
- **취약점 관리**: 의존성 정기 점검(월 1회), 보안 업데이트 지체 없이 반영

### 5.3 물리적 보호조치

- **서버 물리적 관리**: Supabase Cloud (호스팅사 데이터센터)의 물리적 보안 정책에 준함
  - 리전: [[확정 필수, 예: AWS ap-northeast-2 서울]]
  - 접근 통제, 재해복구 등은 호스팅사의 보안 인증(ISO 27001, SOC 2 등)에 의거
- **관리자 단말기**: 지정 단말기에서만 관리 콘솔 접근, 화면 잠금·디스크 암호화 적용

## 6. 이용자 권리 보장

- **동의 거부권**: 위치정보 수집·이용에 동의하지 않을 수 있으며, 이 경우 지도·지역 매칭 기능이 제한됨 (커뮤니티 열람 등 위치 무관 기능은 이용 가능)
- **동의 철회**: 앱 내 설정 → 지역 인증 해제, 또는 서비스 탈퇴를 통해 즉시 철회 가능.
  철회 시 법정 보존분(이용·제공사실 확인자료)을 제외한 개인위치정보를 지체 없이 파기
- **일시 중지 요구권**: 위치정보 수집·이용의 일시적 중지 요구 가능 (앱 설정에서 위치 기능 OFF)
- **개인위치정보 열람·정정·삭제 요구권**: 서면·이메일·앱 문의로 신청 시 지체 없이 처리
- **위치정보 이용·제공사실 확인자료 열람 요구권**: 최근 6개월 이내 열람 가능
- **법정대리인 권리**: 만 14세 미만 이용자는 가입이 차단되어 별도 대응 불요.
  향후 대상 확대 시 법정대리인 동의·권리행사 절차 구축 예정

## 7. 위치정보 제3자 제공 정책

- **원칙**: 이용자의 개인위치정보를 제3자에게 제공하지 않음
- **예외**:
  - 법령에 근거한 요청 (법원 영장, 수사기관 요청 등)
  - 이용자 본인의 명시적 동의가 있는 경우 (제공 시 매회 이용자에게 제공받는 자·일시·목적 즉시 통보)
- **제3자 제공 시 확인자료 보관**: 위치정보법에 따라 6개월 이상 보관
