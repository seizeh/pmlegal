# 운영 점검 주기 (파기 배치 · 정기 점검 · 기한 관리)

- 문서 버전: v1.0 / 작성일: 2026-07-11
- 목적: 위치정보법·개인정보보호법상 **정기 파기·점검 의무**와 **이벤트성 신고 기한**을
  한 곳에 정리하여 운영 중 누락을 방지한다. (사업계획서 5장·처리방침의 보조 문서)

---

## 1. 정기(자동) 파기 배치

DB 자동 파기는 Supabase **pg_cron** 으로 실행한다. 아래 표의 "실행 상태"는 2026-07-11 기준.

| 대상 | 보존/파기 주기 | 근거 | 실행(cron) | 상태 |
|---|---|---|---|---|
| refresh_tokens / rate_limits | 만료·폐기 후 즉시 | 인증 위생 | `auth-cleanup` 매시간(17분) | ✅ 운영 중 |
| 탈퇴자 격리 데이터(app.withdrawn_users) | 30일 후 | 재가입 악용 방지·최소보관 | `withdrawn-users-purge` 매일 03:43 | ✅ 운영 중 |
| **전화 인증코드(phone_verifications)** | 발급 후 1일 | 데이터 최소화(코드 TTL 5분) | `retention-purge` 매일 | ⚠️ **추가 필요** |
| **위치 인증 이력(location_verifications)** | 6개월 후 | 내부 방침(사업계획서 §3.4) | `retention-purge` 매일 | ⚠️ **추가 필요** |
| **사진 인증 로그(photo_verifications, 촬영 좌표 포함)** | 6개월 후 | 위치정보 최소보관 | `retention-purge` 매일 | ⚠️ **추가 필요** |
| 위치정보 이용·제공사실 확인자료 | 6개월 보존 후 파기 | 위치정보법 제16조 제2항 | (로그 테이블 확정 후 배치) | ⛔ 구현 확인 필요 |
| 관리자·취급자 접속기록 | **1년 이상 보관** | 개인정보 안전성 확보조치 기준 | (보관 의무 — 파기 아님) | — |

> ⚠️ 표시 항목은 문서상 약속돼 있으나 현재 자동 배치가 없다. 아래 "부록 A" 마이그레이션으로
> `retention-purge` 배치를 추가하면 문서-현실이 일치한다.

### 백업과 파기의 관계
- **현재(Free 요금제)**: Supabase 자동 백업 미제공 → DB 레코드 삭제 시 **즉시 파기가 완결**.
  단, 수동 백업(pg_dump 등)을 보관하는 경우 그 파일에도 삭제 대상 데이터가 남으므로,
  **수동 백업은 짧은 보존주기(예: 7일)로 회전·폐기**하고 그 사실을 기록한다.
- **정식 오픈 시(Pro 요금제로 업그레이드 예정)**: 자동 일일 백업 **보존 7일** → 삭제된
  개인·위치정보는 최대 7일 내 백업본에서도 소멸.

## 2. 정기 점검 항목 (캘린더)

| 주기 | 항목 | 근거 |
|---|---|---|
| 월 1회 | 의존성/라이브러리 **취약점 점검** 및 보안 업데이트 반영 | 사업계획서 §5.2 |
| 반기 1회 | 위치정보 취급자 **접근권한 재검토**(최소권한 원칙) | 사업계획서 §5.1 |
| 연 1회 | 위치정보 **보호조치 자체 점검** + 취급자 **정기 교육** | 사업계획서 §5.1 |
| 상시 | 신고·1:1 문의 **처리 SLA 7일** 준수 | 사업계획서 §10 |

## 3. 이벤트성(수시) 기한 관리

| 사건 | 기한/조치 | 근거 |
|---|---|---|
| 상호·소재지·주요설비(서버 등) 변경 | 지체 없이 **변경신고** | 위치정보법 제9조 제3항 |
| 휴업/폐업 | **30일 전** 개인위치정보주체 통보 + 파기계획 첨부 신고 | 위치정보법 제11조 |
| 개인정보 유출 인지 | **72시간 내** 정보주체 통지·전문기관 신고 | 개인정보보호법 |
| 개인위치정보 침해/오남용 신고 접수 | 지체 없이 확인·조치·통지 | 위치정보법 |
| 손해배상책임 보장(보험·공제) 요건 해당 여부 | **매년** 확인, 해당 시 이행 | 개인정보보호법 |

## 부록 A. 파기 배치 추가 마이그레이션(제안)

아래를 적용하면 ⚠️ 3개 항목이 매일 자동 파기된다. (pmdb 마이그레이션으로 배포)

```sql
-- 위치정보/인증코드 보존기간 경과분 자동 파기
create or replace function app.cleanup_retention()
returns void language sql security definer set search_path to '' as $$
  -- 전화 인증코드(OTP): 발급 1일 경과분 파기(코드 TTL 5분, 데이터 최소화)
  delete from public.phone_verifications  where created_at < now() - interval '1 day';
  -- 위치 인증 이력: 6개월 경과분 파기(사업계획서 §3.4)
  delete from public.location_verifications where created_at < now() - interval '6 months';
  -- 사진 인증 로그(촬영 좌표 포함): 6개월 경과분 파기
  delete from public.photo_verifications  where created_at < now() - interval '6 months';
$$;

-- 매일 03:23 실행
select cron.schedule('retention-purge', '23 3 * * *', $$select app.cleanup_retention();$$);
```

> 주의: `photo_verifications` 행 삭제는 스토리지 이미지 파일을 지우지 않는다(이미지는 게시글
> 수명과 함께 관리). 촬영 좌표 등 위치 메타데이터 로그만 파기된다.
